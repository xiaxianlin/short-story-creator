# 短篇小说创作项目

这是一个基于 Codex 的短篇小说创作辅助项目，通过自定义技能（Skills）提供结构化的小说创作和审稿工作流。

## 可用技能

### 1. short-novel-workflow（小说创作）

**触发方式**：用户说"创作小说"、"开始创作"、"短篇小说创作"、"帮我写一个故事"

**工作流程**（11步）：
1. 确定核心立意（遵循“单一聚焦”原则）
2. 选择受众（男频/女频）
3. 填写创作想法
4. 选择题材（历史、玄幻、都市、奇幻、仙侠、科幻、言情、悬疑、现实等）
5. 设置目标字数（推荐3,000-50,000字）
6. 生成故事背景设定
7. 生成角色设定（展示核心特征，冰山理论）
8. 生成情节大纲（三幕式/起承转合结构）
9. 生成小说名称
10. 按章节创作内容（运用“展示而非讲述”原则）
11. 保存小说到 `/novels/[小说名称]/` 目录

### 2. manuscript-reviewer（小说审稿）

**触发方式**：用户说"审稿"、"审阅稿件"、"评估小说"、"审读文章"、"这个小说写得好不好"

**工作流程**：
1. 确定审稿方向类别（现实主义/推理悬疑/奇幻/言情等）
2. 格式初审
3. AI检测（五大特征：华美空洞、逻辑矛盾、节奏割裂、情节停滞、情感缺失，并结合 anti-ai-writing 识别 AI 腔与模板句式）
4. 首段评估（前400字决定命运）
5. 12维度深度评分（影响力22%、创意性15%、语言15%等权重）
6. 撰写审稿意见
7. 保存审稿报告到 `novels/{小说名称}/reports/审稿报告_{日期}.md`

**决策阈值**：
- ≥8分：建议录用
- 6-8分：修改后重审
- <6分：退稿

### 3. anti-ai-writing（反 AI 味写作）

**触发方式**：用户说"去掉AI味"、"改得像人写的"、"检查AI腔"、"降低模板感"、"润色得自然一点"

**工作流程**：
1. 判断文本类型（小说正文/故事梗概/审稿意见/通用文案）
2. 识别高风险问题（抽象词、模板句式、过度解释、人物同腔）
3. 按“问题-原因-改法”做语言诊断
4. 在不改核心信息和设定的前提下重写
5. 结合《短篇小说创作理论》微调节奏和语气（针对文学性或悬疑感等风格优化）

## 文件保存规范

### 小说创作保存位置
```
novels/{小说名称}/
├── 《小说名称》.md       # 完整正文（必须）
├── 设定集.md             # 背景、角色设定
├── 大纲.md               # 情节大纲、章节规划
└── reports/              # 审稿报告目录（可选）
    └── 审稿报告_YYYY-MM-DD.md
```

### 命名规范
- 小说文件名：`《{小说名称}》.md`
- 审稿报告：`审稿报告_YYYY-MM-DD.md`

## 创作规范

- 目标字数：默认 20,000 字（可调整范围 10,000-50,000 字）
- 章节长度：每章 2,000-3,000 字
- 支持题材：历史、玄幻、都市、奇幻、仙侠、科幻、言情、悬疑、现实、体育、其他
- 受众类型：男频、女频

## 快捷命令

用户在创作过程中可随时使用：
- `重做` - 重新生成当前步骤
- `修改` - 修改之前的设定
- `进度` - 查看当前进度
- `预览` - 预览已创作内容
